Документация

Как добавить навык

01

Создать свой

Директория в skills/my_skill/ с тремя файлами:

skills/my_skill/
  skill.yaml
  skill.py
  README.md
# skill.yaml
name: my_skill
version: "1.0.0"
description: "Что делает навык"
tools:
  - my_tool
02

Импорт из проекта

Скопируйте навык из другого проекта My Agent:

cp -r other-project/skills/cool_skill \
   my-agent/skills/

Зарегистрируйте:

agent add-skill universal cool_skill
03

Типы навыков

static      # YAML + Python код
dynamic     # Генерируется LLM
composite   # Объединяет другие
external    # Внешний API/MCP

Навыки наследуют контекст от агента и могут вызывать друг друга.

Преимущества

Почему My Agent

01

Dual-provider с умным fallback

NeuroAPI (~1.5с) как основной, OpenRouter (~6с) как запасной. При перегрузке система делает 3 попытки с задержками 10, 20, 30 секунд, затем переключается. ~95% запросов укладываются в 2 секунды.

02

Async-native без блокировок

Каждый слой полностью асинхронный. Ни одного run_in_executor. Параллельное выполнение инструментов через asyncio.gather() ускоряет задачи в 3-5 раз.

03

Docker-only sandbox

Выполнение кода только в Docker: --network none --memory 512m --read-only. Нет fallback на subprocess. Если Docker недоступен — система отказывает.

04

Экономия токенов с skill cache

Предварительная фильтрация отбирает только релевантные навыки. Из 57 инструментов остается ~12. Экономия ~30% токенов на каждый запрос.

05

Dual-database

Dev — SQLite (zero-config). Production — PostgreSQL с pooling. Автоопределение по DATABASE_URL. Alembic миграции, WAL mode, FTS5.

06

Автономность

APScheduler с persistence — задачи переживают перезапуск. Self_dev позволяет агенту читать и модифицировать свой код с human approval по умолчанию.
